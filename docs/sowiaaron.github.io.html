Quarto

Quarto ermöglicht ausführbaren Code und Text zusammen in einem Dokument zu verwalten. Dies wird mit dem Begriff weave bzw. weaving bezeichnet. Um mehr darüber und über Quarto zu erlernen schau auf die Website: https://quarto.org.

Ein Quellcode unterschiedliche Formate

Aus der gleichen Datei wird die PDF und die HTML Version erzeugt. Der Parameter URL wird im YML Header dieses Dokuments mit dem Titel sowiaaron.github.io

gesetzt. Und kann nun z. B. in R Syntax verwandt werden.

Code 
params$url
[1] "sowiaaron.github.com"

Über sowiaaron.github.com kann hier der Parameter URL ausgegeben werden sowiaaron.github.com.

Dieser wird in der PDF Version verwadnt um Ihn z.B in einem QR Code anzuzeigen. Dieser Text ist nur in der HTML Version zu sehen. In der PDF Variante wird die URL als QR Code an dieser Stelle eingefügt.

In der nächsten Version soll der Parameter URL in eine Projektweite Datei _variables.yml verschoben werden. Dabei soll auch die Behandlung der Parameter beim erzeugen der PDF Version mit Latex geändert werden.

Aktuell wird der Parameter in r ausgegegeben und in der Latex Umgebung dem Befehl psbarcode übergeben.

Code 
\begin{pspicture}(4,4) 
\psbarcode[linecolor=blue]{`r .QuartoInlineRender(params$url)`}{includetext}{qrcode} 
\end{pspicture}
